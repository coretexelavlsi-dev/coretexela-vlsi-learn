Lesson 006 – Data Representation & Ranges

Level: Beginner | Stage: Digital Electronics | Duration: 10–15 minutes

Concept Overview

Data representation defines how numbers are stored and processed inside digital systems using a fixed number of bits. Since hardware resources are limited, every digital value has a specific range it can represent.

Understanding data representation helps prevent overflow, underflow, and logic errors in digital and VLSI designs.

Unsigned Data Representation

In unsigned representation, all bits are used to represent the magnitude of the number.

For an n-bit unsigned number:
Range = 0 to (2ⁿ − 1)

Example (4-bit unsigned):
0000 = 0
1111 = 15

Signed Data Representation (2’s Complement)

Signed numbers allow representation of both positive and negative values. The most widely used method is 2’s complement.

For an n-bit signed number:
Range = −2ⁿ⁻¹ to (2ⁿ⁻¹ − 1)

Example (4-bit signed):
1000 = −8
0111 = +7

Signed vs Unsigned Comparison

Type Range Typical Use
Unsigned 0 to 2ⁿ − 1 Counters, addresses
Signed −2ⁿ⁻¹ to 2ⁿ⁻¹ − 1 Arithmetic operations

Overflow and Underflow

Overflow occurs when the result of an operation exceeds the maximum value that can be represented with the given bit-width.

Example (4-bit unsigned):
1111 + 0001 → overflow

Underflow occurs when the result goes below the minimum representable value.

Why Ranges Matter in VLSI

  • Register width selection
  • ALU and datapath design
  • HDL signal declarations
  • Preventing functional bugs

Key Takeaways

  • Digital data uses fixed bit-width representation
  • Signed and unsigned numbers have different ranges
  • 2’s complement is the industry standard
  • Overflow and underflow are real hardware issues

Self-Test

1. What is the range of an 8-bit unsigned number?

2. What is the range of an 8-bit signed number?

3. Why does overflow occur?

Self-Test Solutions

Answer 1: 0 to 255

Answer 2: −128 to +127

Answer 3: Because the result exceeds the fixed bit-width
